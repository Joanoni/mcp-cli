# Architecture Overview

## What is cli-runner?

**cli-runner** is an MCP (Model Context Protocol) server that reads JSON configuration files describing CLI tools and dynamically registers each configured subcommand as an MCP tool. An MCP client (such as Claude Desktop or Roo) can then invoke those tools to execute real CLI commands on the host machine and receive their output.

## High-Level Flow

```
src/cli/*.json
     │
     │  (develop & validate locally)
     ▼
mcp/cli-runner/clis/*.json
     │
     │  (loaded at runtime by the server)
     ▼
cli-runner MCP Server  (mcp/cli-runner/dist/index.js)
     │
     │  (stdio transport — JSON-RPC 2.0)
     ▼
MCP Client  (Claude Desktop, Roo, etc.)
```

1. **Develop** — Author new CLI config JSONs in `src/cli/` and validate them against the schema.
2. **Promote** — Copy validated JSONs to `mcp/cli-runner/clis/` to make them available in production.
3. **Runtime** — The server reads `mcp/cli-runner/config.json` to locate the `clis/` directory, loads every `.json` file found there, and registers one MCP tool per `commands` entry.
4. **Invocation** — The MCP client calls a tool by name; the server spawns the corresponding CLI process, captures stdout/stderr, and returns the result as text content.

## Directory Roles

| Path | Role |
|---|---|
| `src/mcp/cli-runner/` | TypeScript source code for the MCP server |
| `src/mcp/cli-runner/src/` | Application source files (`index.ts`, `config.ts`, `executor.ts`, `tool-registry.ts`) |
| `src/cli/` | Staging area — develop and test new CLI config JSONs here before promoting to production |
| `mcp/cli-runner/` | Production build root — the directory pointed to by MCP client configuration |
| `mcp/cli-runner/dist/` | Compiled JavaScript output (generated by `npm run build`) |
| `mcp/cli-runner/clis/` | Production CLI config JSONs consumed by the running server |
| `mcp/cli-runner/config.json` | Global server configuration (`defaultTimeout`, `clisDirectory`) |
| `scripts/` | Utility scripts for workspace maintenance |

## Key Components

| Source File | Responsibility |
|---|---|
| [`src/index.ts`](../src/mcp/cli-runner/src/index.ts) | Entry point — bootstraps the server, loads config, connects stdio transport |
| [`src/config.ts`](../src/mcp/cli-runner/src/config.ts) | Zod schemas, type definitions, and JSON loaders for global and CLI configs |
| [`src/tool-registry.ts`](../src/mcp/cli-runner/src/tool-registry.ts) | Iterates CLI configs and registers one `server.tool()` per subcommand |
| [`src/executor.ts`](../src/mcp/cli-runner/src/executor.ts) | Spawns child processes, captures output, enforces timeouts |
