## `src/` vs `mcp/` Separation

| Aspect | `src/` | `mcp/` |
|---|---|---|
| **Purpose** | Source code and development artifacts | Production build output served to MCP clients |
| **Contents** | TypeScript source files, draft CLI subdirectories | Compiled JavaScript, validated CLI subdirectories |
| **Edited by** | Developers | Generated by `tsc` (JS files) or manually promoted (CLI subdirectories) |
| **Committed** | Yes — source of truth | Yes — so clients can run without a build step |
| **MCP client points here?** | No | Yes — `mcp/cli-runner/dist/index.js` |

The separation ensures that:
- The MCP client always runs stable, compiled JavaScript from `mcp/cli-runner/dist/`.
- TypeScript compilation errors in `src/` never affect a running production server.
- CLI config subdirectories can be updated in `mcp/cli-runner/clis/` independently of a full TypeScript rebuild.

## Key Configuration Files

| File | Purpose |
|---|---|
| [`src/mcp/cli-runner/tsconfig.json`](../../src/mcp/cli-runner/tsconfig.json) | TypeScript compiler options; sets `outDir` to `mcp/cli-runner/dist/` |
| [`src/mcp/cli-runner/package.json`](../../src/mcp/cli-runner/package.json) | Dev dependencies and build/dev scripts |
| [`mcp/cli-runner/package.json`](../../mcp/cli-runner/package.json) | Runtime dependencies; `main` points to `dist/index.js` |
| [`mcp/cli-runner/config.json`](../../mcp/cli-runner/config.json) | Global server config read at startup |
