# Build and Deploy

## How TypeScript Compilation Works

The TypeScript source in `src/mcp/cli-runner/src/` is compiled using the TypeScript compiler (`tsc`) configured by [`src/mcp/cli-runner/tsconfig.json`](../../src/mcp/cli-runner/tsconfig.json).

### Compiler Settings Summary

| Option | Value | Effect |
|---|---|---|
| `target` | `ES2020` | Output JavaScript compatible with Node.js v14.11+ |
| `module` | `commonjs` | Emit `require()`-style modules (Node.js default) |
| `outDir` | `../../../mcp/cli-runner/dist` | Compiled output goes to `mcp/cli-runner/dist/` |
| `rootDir` | `./src` | Source root; preserves directory structure in output |
| `strict` | `true` | Enables all strict type-checking flags |
| `declaration` | `true` | Emits `.d.ts` type declaration files alongside JS |
| `sourceMap` | `true` | Emits `.js.map` files for debugging |

## Running the Build

```bash
cd src/mcp/cli-runner
npm run build
```

This executes `tsc` with no additional arguments, using `tsconfig.json` in the same directory.

**Expected output on success:** no output (tsc is silent on success).

**On error:** TypeScript compiler errors are printed to stdout with file paths and line numbers. Fix the errors in `src/mcp/cli-runner/src/` and re-run.

### Development Mode (no build required)

During development you can run the server directly from TypeScript source using `ts-node`:

```bash
cd src/mcp/cli-runner
npm run dev
```

This executes `ts-node src/index.ts`. Note that `ts-node` resolves the config path relative to the source location, so the server will still look for `mcp/cli-runner/config.json` at runtime.

## Updating Production CLI Configs

CLI config JSON files in `mcp/cli-runner/clis/` are **not** affected by the TypeScript build. They are managed independently:

| Action | Command |
|---|---|
| Add a new CLI config | Copy from `src/cli/<tool>.json` to `mcp/cli-runner/clis/<tool>.json` |
| Edit an existing config | Edit `mcp/cli-runner/clis/<tool>.json` directly |
| Remove a CLI config | Delete the file from `mcp/cli-runner/clis/` |

After any change to `clis/`, **restart the MCP server** (or the MCP client) to reload the configs. No rebuild is needed for JSON-only changes.

See [docs/cli-runner/adding-clis.md](../cli-runner/adding-clis.md) for the full workflow.

## Full Update Workflow

When you modify TypeScript source **and** CLI configs:

```bash
# 1. Make changes to TypeScript source in src/mcp/cli-runner/src/
# 2. Rebuild
cd src/mcp/cli-runner && npm run build

# 3. Optionally update CLI configs
copy src\cli\<tool>.json mcp\cli-runner\clis\<tool>.json

# 4. Restart the MCP client / server process
```

When you only update CLI configs (no TypeScript changes):

```bash
# 1. Edit or copy JSON files into mcp/cli-runner/clis/
# 2. Restart the MCP client / server process
# (no build step needed)
```

## Runtime Dependencies

The production server (`mcp/cli-runner/`) has its own [`package.json`](../../mcp/cli-runner/package.json) with only runtime dependencies:

```json
{
  "dependencies": {
    "@modelcontextprotocol/sdk": "^1.10.2",
    "zod": "^3.24.2"
  }
}
```

If deploying to a new machine, install these before starting the server:

```bash
cd mcp/cli-runner
npm install
node dist/index.js
```

The `devDependencies` (TypeScript, ts-node, `@types/node`) live only in `src/mcp/cli-runner/package.json` and are never needed at runtime.
